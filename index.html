    Make `sprite.json` by hand in text-editor 

    {
  "iconname" : {
    "width": 32,
    "height": 32,
    "x" : 0,
    "y": 0,
    "pixelRatio": 1
  }
}

    Sprite spec by Mapbox

    A value of 1 indicates a classic 96 DPI (76 DPI on some platforms) display, while a value of 2 is expected for HiDPI/Retina displays. ow can we solve this problem with high DPI devices? Basically, provide two different version of each image. all fixed size graphics needs to be doubled in width and height in order to keep the sharpness of the graphic.
